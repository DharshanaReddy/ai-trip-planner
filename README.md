## Tech Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| Agents | **LangGraph** | Multi-agent orchestration with typed state |
| LLM | **OpenAI GPT-4o-mini** | Reasoning across all 4 agents |
| RAG | **ChromaDB + LangChain** | Travel knowledge retrieval |
| Observability | **LangSmith** | Agent tracing, latency, token usage |
| Backend | **FastAPI** | Async REST API with in-memory caching |
| Frontend | **Streamlit + Plotly** | Interactive UI with budget analytics |
| Cache | **Redis** | Response caching by destination hash |
| Infrastructure | **Docker + Compose** | Containerized multi-service deployment |

## Features

- **4-Agent Pipeline** вҖ” ResearchAgent вҶ’ ItineraryAgent вҶ’ BudgetAgent вҶ’ ReviewAgent
- **RAG-enhanced research** вҖ” ChromaDB retrieves relevant travel knowledge before LLM generation, reducing hallucination
- **LangSmith tracing** вҖ” Every agent call observable with latency, token count, and chain visualization
- **Budget analytics** вҖ” Donut chart breakdown, daily cost bar chart, budget gauge with variance
- **Prompt-level caching** вҖ” Same destination/params served from cache (SHA-256 key)
- **Structured output** вҖ” Pydantic-validated schemas at every pipeline stage

## Quick Start

```bash
git clone https://github.com/DharshanaReddy/ai-trip-planner
cd ai-trip-planner
cp .env.example .env  # add your API keys
```

**Option A вҖ” Docker (recommended):**
```bash
docker-compose up --build
# Frontend: http://localhost:8501
# Backend:  http://localhost:8000/docs
```

**Option B вҖ” Local:**
```bash
pip install -r requirements.txt

# Terminal 1 вҖ” Backend
uvicorn backend.main:app --port 8000 --reload

# Terminal 2 вҖ” Frontend
cd frontend && streamlit run app.py
```

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `OPENAI_API_KEY` | вң… | OpenAI API key |
| `LANGCHAIN_API_KEY` | Recommended | LangSmith API key for tracing |
| `LANGCHAIN_PROJECT` | No | LangSmith project name (default: `ai-trip-planner`) |
| `REDIS_URL` | No | Redis for production caching |
| `BACKEND_URL` | No | Frontend вҶ’ Backend URL |


## LangSmith Observability

Each trip generation creates a traced run in LangSmith showing:
- Per-agent latency breakdown
- Token consumption at each node
- Full input/output for debugging
- Chain visualization

Get a free LangSmith key at [smith.langchain.com](https://smith.langchain.com)
